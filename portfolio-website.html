Project Overview

The Challenge

Build a responsive portfolio website to showcase my UX portfolio using my knowledge of HTML, CSS, and Javascript.

The Goal

Showcase my portfolio projects in a way that communicates my design process across multiple platforms, and make my contact information and resume readily available to potential employers.

Project Duration

2 Months

Tools Used

  • VSCode
  • GitHub
  • Adobe Illustrator
  • Adobe Photoshop

Information Architecture

The best way divide the content of the website up was into four categories: home page, about page, project pages, and finally a contact page. Once I determined what each page would consist of, I began mapping out where the content would live based on the wireframes provided by CareerFoundry. Before diving into the coding phase of the process, I created a style guide for the color scheme and fonts based on my branding.

Color Scheme

The color scheme I selected represents the same values my branding holds: clean, simple, and professional, while maintaining AA accessibility standards.

Font Choice

Like the color scheme, I chose Roboto: a font that would best represent my branding and remain easy to read regardless of the size.

Development Phase

After establishing a basis for the styling of my website, I dove right into HTML where I plugged all content into the site. Once all the content was completed, I began creating developing layouts for each media query starting with mobile first, then desktop and tablet. One challenge was creating a navigation menu that would function at both mobile and desktop screen sizes. To do this, I utilized my knowledge of Javascript to create a hamburger menu for mobile screen sizes.

Usability Testing

Once the website was ready to be tested, I recruited 5 participants who best fit my target audience. Each participant performed four tasks based on what an employer would likely do or look for when visiting the website:

Scenario #1:

You have just landed on the home page for the first time. Explain your first impression and any visual elements you noticed. (SEQ not applicable).

Scenario #2:

You scroll through the projects and PlutoPay catches your eye. Where can you find additional information about this project? What was the project about?

Scenario #3:

After viewing all of the projects you become interested to learn about this person's education and previous experience. As a result, you want to look at their resume. Where can you find it?

Scenario #4:

You have now read through their resume and are impressed with their skills and background. You'd like to reach out to them to talk about the job opening. How do you proceed?

Usability Test Results

After performing each task, the participant was asked to rate the difficulty of the task using an SEQ scale. Most tasks took the user under one minute each, and overall the tests were a success. The SEQ averages ranged from 6.67 to 7 out of 7, which is a very high score.

Potential adjustments were rated on a scale from 0-4 to determine the importance of making said change, 0 being the least crucial changes and 4 being the most crucial.

Revisions

Notable changes made to the website include adding an interactive table of contents to the PlutoPay project page. This betters the user experience greatly, as the project page is extensive. Having an interactive table of contents allows for easier navigation on the page. Minor changes were made to improve the appeal such as fixing the padding issues in various project pages.

Next Steps & Reflection

My portfolio website represents who I am as a UI/UX Designer: always improving. I plan to continue pushing my website further by focusing on visual appeal and formatting the content to be easier to digest for users. Other projects will be added as well, similar to this one. As I learn new UI/UX skills, deepen my knowledge of code, and grow as a designer I will come back to this website to ensure it accurately depicts who I am through iterative design.
